Если в поле нужно вводить только число, то для него указывается атрибут data-numbers-only.

Спойлер элементы:
    1. Родительскому блоку задается класс "spoiler";
    2. Скрываемомым элементам задается класс "spoiler__hideable";
    3. Кнопке, показывающей скрываемые элементы, задается класс "spoiler__button".
--------------------------------------------------------------------------------------------------------
Проверка на заполненность поля:
    1. Чтобы указать минимальную длину поля, нужно к input прописать атрибут data-complete-length="a, b", где a и b - числа. Пример: data-complete-length="5, 25" значит, что длина значения должна быть от 5 до 25 символов.
    Длина значения (value) этого поля в символах должна быть больше или равна "a" и меньше или равна "b". Если указать только число "a" и не указывать "b", то длина поля просто должна быть больше или равна "a", то есть число "b" указывать НЕ обязательно.
    2. Если указано поле data-mask (см. в "Input'ы с шаблонами (масками)"), data-complete-length указвать не нужно, оно привязано к количеству символов "_", указанных в data-mask.
--------------------------------------------------------------------------------------------------------
Input'ы с шаблонами (масками) (например, номер телефона):
    1. Маска указывается в data-mask;
    2. Места, которые пользователи будут заполнять символами обозначаются "_". Пример: "+7 (___) - ___ - __ - __" - здесь будут заменяться только символы "_", а символы "+7", "(", ")" и "-", а также пробелы будут оставаться на месте;
    3. placeholder выставится автоматически после инициализации скрипта, если не указан.
--------------------------------------------------------------------------------------------------------
Добавляемые блоки через кнопку (пример: "место работы"):
    1. Создается внутри блока ".forms__container";
    2. Для указания максимального количества полей используется data-add-max к элементу ".add-field";
    3. Добавляемые блоки помечаются атрибутом data-add-group. Дальше указывается обычная верстка блока;
    4. Если указать data-add-group="skip:x", где x = число, элемент в первых блоках x блоках не появится, а будет только в последующих. Если x < 0, то такой элемент наоборот появится только в первых x блоках.
-------------------------------------------------------------------------------------------------------- 
Добавляемые поля через чекбокс (пример: "способ связи"):
    1. Для указания чекбокса, который добавляет поля, необходимо ему присвоить data-add-group-checkbox="name", где name должно быть названием поля !ВАЖНО: name не должен содержать знак двоеточия ":"!;
    2. Указанный в 1. name нужно подставить создаваемому элементу data-add-group="name". Элемент будет добавлен после этого чекбокса, когда чекбокс будет выбран;
